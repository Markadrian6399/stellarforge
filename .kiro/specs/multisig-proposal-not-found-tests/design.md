# Design Document: multisig-proposal-not-found-tests

## Overview

This feature adds three Rust unit tests inside the existing `#[cfg(test)] mod tests` block in `contracts/forge-multisig/src/lib.rs`. Each test initialises a fresh 2-of-3 multisig (reusing the existing `setup_2of3` helper), then calls `try_approve`, `try_reject`, or `try_execute` with proposal ID `9999` — a value that has never been stored — and asserts the result is `Err(Ok(MultisigError::ProposalNotFound))`. No new files, no new dependencies, and no changes to production code are required.

## Architecture

The tests live entirely within the existing test module. They follow the same pattern used throughout the file:

```
setup_2of3 → mock_all_auths → try_<fn>(&owner, &9999_u64) → assert_eq!(result, Err(Ok(MultisigError::ProposalNotFound)))
```

No new modules, traits, or abstractions are introduced.

## Components and Interfaces

### Existing helpers used

| Symbol | Location | Purpose |
|---|---|---|
| `setup_2of3` | `lib.rs` test module | Creates a registered contract + client + 3 owner addresses, initialised as 2-of-3 |
| `MultisigContractClient::try_approve` | generated by `contractimpl` | Returns `Result<Result<(), MultisigError>, InvokeError>` |
| `MultisigContractClient::try_reject` | generated by `contractimpl` | Same shape |
| `MultisigContractClient::try_execute` | generated by `contractimpl` | Same shape |
| `MultisigError::ProposalNotFound` | `lib.rs` | Discriminant 4; the expected error variant |

### Test functions to add

- `test_approve_non_existent_proposal_returns_not_found`
- `test_reject_non_existent_proposal_returns_not_found`
- `test_execute_non_existent_proposal_returns_not_found`

## Data Models

No new data models. The tests operate on the existing `Proposal` storage key `DataKey::Proposal(proposal_id: u64)`. Using `9999` as the proposal ID is sufficient because `setup_2of3` never calls `propose()`, so persistent storage contains no `Proposal` entries.

## Correctness Properties

*A property is a characteristic or behavior that should hold true across all valid executions of a system — essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees.*

Property 1: approve() on a non-existent proposal returns ProposalNotFound
*For any* freshly initialised MultisigContract and any proposal ID that has never been stored, calling `approve()` with that ID returns `Err(MultisigError::ProposalNotFound)` and does not panic.
**Validates: Requirements 1.1, 1.2**

Property 2: reject() on a non-existent proposal returns ProposalNotFound
*For any* freshly initialised MultisigContract and any proposal ID that has never been stored, calling `reject()` with that ID returns `Err(MultisigError::ProposalNotFound)` and does not panic.
**Validates: Requirements 2.1, 2.2**

Property 3: execute() on a non-existent proposal returns ProposalNotFound
*For any* freshly initialised MultisigContract and any proposal ID that has never been stored, calling `execute()` with that ID returns `Err(MultisigError::ProposalNotFound)` and does not panic.
**Validates: Requirements 3.1, 3.2**

## Error Handling

The tests themselves are the error-handling verification. The production code already uses `.ok_or(MultisigError::ProposalNotFound)` on the persistent storage read in all three functions; the tests confirm that path is reachable and returns the correct variant rather than panicking.

## Testing Strategy

### Approach

Because the Soroban SDK does not ship a property-based testing library and the contract is compiled to Wasm, we use concrete example-based tests. Each test exercises a single, deterministic scenario (proposal ID `9999` on a fresh contract) which is sufficient to cover the error path. The no-panic guarantee is implicit: `try_*` methods return `Result` and never unwind.

### Test structure (Rust)

```rust
#[test]
fn test_approve_non_existent_proposal_returns_not_found() {
    let env = Env::default();
    env.mock_all_auths();
    let (client, o1, _, _) = setup_2of3(&env);
    let result = client.try_approve(&o1, &9999_u64);
    assert_eq!(result, Err(Ok(MultisigError::ProposalNotFound)));
}

#[test]
fn test_reject_non_existent_proposal_returns_not_found() {
    let env = Env::default();
    env.mock_all_auths();
    let (client, o1, _, _) = setup_2of3(&env);
    let result = client.try_reject(&o1, &9999_u64);
    assert_eq!(result, Err(Ok(MultisigError::ProposalNotFound)));
}

#[test]
fn test_execute_non_existent_proposal_returns_not_found() {
    let env = Env::default();
    env.mock_all_auths();
    let (client, o1, _, _) = setup_2of3(&env);
    let result = client.try_execute(&o1, &9999_u64);
    assert_eq!(result, Err(Ok(MultisigError::ProposalNotFound)));
}
```
